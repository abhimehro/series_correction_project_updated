## 2024-06-01 - Avoid incremental DataFrame builds for Gap Filling
**Learning:** Building small pandas DataFrames in a loop inside `correct_gaps` using `pd.DataFrame()` incrementally causes massive object instantiation overhead, slowing down gap filling logic substantially (~30x slower than purely vectorized logic in microbenchmarks).
**Action:** When creating variable-sized segments to merge (e.g., timestamps for missing gap periods), append the raw index/numpy arrays to a list and use `np.concatenate()` to create a single array outside the loop. Construct the `pd.DataFrame` containing these concatenated elements precisely once before appending to the main DataFrame.
